## Business Model

**Syntaxis Labs** is an independent commission-only sales representative, not a lead-generation agency. The correct workflow is:

1. Source opportunities on CommissionCrowd (or other public listing platforms).
2. Score them for fit as a rep.
3. Apply to the **vendor/principal** to represent them.
4. Only after **accepted/onboarded**, build ICP-targeted buyer-outreach campaigns.

**Buyer outreach before vendor acceptance is premature and blocked.**

## Opportunity Pipeline Stages

| `sourced` | Discovered on listing | Yes (automated ingestion) |

| `application_draft_created` | Application-to-principal draft written | No — requires `apply_to_principal` approval |
| `application_approved` | Operator approved applying | No — operator updates Sheet manually |
| `application_submitted` | Application sent to vendor | No — requires `application_submitted` approval |
| `accepted` | Vendor accepted; onboarding complete | No — vendor response, operator confirms |
| `rejected` | Vendor declined; opportunity closed | No — vendor response, operator confirms |
| `icp_campaign_ready` | Buyer-outreach campaign drafted | No — requires `icp_campaign_draft` approval |
| `selling_active` | Buyer outreach (ICP) in progress | No — requires `icp_campaign_send` approval |

## Approval Taxonomy

| approval_action | Gates | Risk default |
|---|---|---|
| `research_scoring` | Scoring a newly sourced lead | low |
| `deeper_research` | Public read-only deeper research | medium |
| `apply_to_principal` | Creating and reviewing an application-to-principal draft | medium |
| `application_submitted` | Actually submitting the application to the vendor | medium |
| `icp_campaign_draft` | Drafting a buyer-outreach campaign for the ICP | low |
| `icp_campaign_send` | Sending buyer-outreach messages | low |

---

## Hard Constraints (Invariant)

1. **No buyer outreach before vendor acceptance.** If `OpportunityStage` is below `accepted`, any `icp_campaign_*` or `outreach_send` action is **blocked**.
2. **No application without approval.** `application_submitted` requires `apply_to_principal` approval first.
3. **No secrets printed.** Every log, report, and notification must be safe.
4. **No CommissionCrowd login/API use without explicit operator approval.**
5. **Every factual claim must include source/provenance.** Unverified claims must be marked.
6. **Approval statuses are never modified automatically.** Only the operator updates the Sheet.
